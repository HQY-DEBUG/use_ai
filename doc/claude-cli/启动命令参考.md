## 概述

本文档列出 `claude` 命令在**启动阶段**（命令行中直接传入）可用的子命令与标志，不含会话内斜杠命令。

---

## 子命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `claude` | 启动交互式会话 | `claude` |
| `claude "query"` | 携带初始提示词启动交互式会话 | `claude "解释这个项目"` |
| `claude update` | 更新到最新版本 | `claude update` |
| `claude auth login` | 登录 Anthropic 账户，支持 `--email` 预填邮箱、`--sso` 强制 SSO | `claude auth login --email user@example.com --sso` |
| `claude auth logout` | 登出 Anthropic 账户 | `claude auth logout` |
| `claude auth status` | 以 JSON 格式显示认证状态，`--text` 获取人类可读输出 | `claude auth status --text` |
| `claude agents` | 列出所有已配置的子代理（按来源分组） | `claude agents` |
| `claude mcp` | 配置 Model Context Protocol (MCP) 服务器 | `claude mcp` |
| `claude remote-control` | 启动 Remote Control 服务器，允许从 Claude.ai 控制本地实例 | `claude remote-control --name "My Project"` |

## 启动标志

### 会话控制

| 标志 | 说明 | 示例 |
|------|------|------|
| `--continue`, `-c` | 加载当前目录中最近的对话 | `claude -c` |
| `--resume`, `-r` | 按 ID 或名称恢复特定会话，或显示交互式选择器 | `claude -r "auth-refactor"` |
| `--fork-session` | 恢复时创建新会话 ID，而非重用原 ID | `claude -r abc123 --fork-session` |
| `--from-pr` | 恢复链接到特定 GitHub PR 的会话（接受 PR 号或 URL） | `claude --from-pr 123` |
| `--name`, `-n` | 为会话设置显示名称 | `claude -n "my-feature-work"` |
| `--session-id` | 为对话指定特定会话 ID（须为有效 UUID） | `claude --session-id "550e8400-..."` |
| `--init` | 运行初始化 hooks 并启动交互模式 | `claude --init` |
| `--init-only` | 运行初始化 hooks 后退出，不进入交互会话 | `claude --init-only` |
| `--maintenance` | 运行维护 hooks 后退出 | `claude --maintenance` |

### 模型与工作量

| 标志 | 说明 | 示例 |
|------|------|------|
| `--model` | 设置本次会话使用的模型（支持别名 `sonnet`/`opus` 或完整名称） | `claude --model claude-sonnet-4-6` |
| `--effort` | 设置工作量级别：`low`/`medium`/`high`/`max`（仅 Opus 4.6） | `claude --effort high` |
| `--betas` | 指定 Beta 特性标头（仅限 API Key 用户） | `claude --betas interleaved-thinking` |
| `--fallback-model` | 默认模型过载时自动回退至指定模型（仅打印模式） | `claude -p --fallback-model sonnet "query"` |

### 系统提示

| 标志 | 说明 | 示例 |
|------|------|------|
| `--system-prompt` | 用自定义文本**替换**整个系统提示 | `claude --system-prompt "你是 Python 专家"` |
| `--system-prompt-file` | 从文件加载系统提示（替换默认提示），与 `--system-prompt` 互斥 | `claude --system-prompt-file ./prompts/review.txt` |
| `--append-system-prompt` | 将自定义文本**附加**到默认系统提示末尾 | `claude --append-system-prompt "始终使用 TypeScript"` |
| `--append-system-prompt-file` | 从文件加载内容附加到默认系统提示 | `claude --append-system-prompt-file ./style-rules.txt` |

> 附加标志（`--append-*`）可与替换标志组合使用。大多数场景推荐用附加标志，以保留 Claude Code 内置功能。

### 工具与权限

| 标志 | 说明 | 示例 |
|------|------|------|
| `--tools` | 限制 Claude 可使用的内置工具，`""` 禁用全部，`"default"` 为全部 | `claude --tools "Bash,Edit,Read"` |
| `--allowedTools` | 无需权限提示即可执行的工具列表 | `claude --allowedTools "Read" "Bash(git log *)"` |
| `--disallowedTools` | 从上下文中移除的工具，不可使用 | `claude --disallowedTools "Edit"` |
| `--permission-mode` | 以指定权限模式启动 | `claude --permission-mode plan` |
| `--dangerously-skip-permissions` | 跳过所有权限提示（谨慎使用） | `claude --dangerously-skip-permissions` |
| `--allow-dangerously-skip-permissions` | 将权限绕过作为可选项启用，而不立即激活 | `claude --permission-mode plan --allow-dangerously-skip-permissions` |
| `--permission-prompt-tool` | 指定 MCP 工具处理非交互模式下的权限提示 | `claude -p --permission-prompt-tool mcp_auth_tool "query"` |
| `--disable-slash-commands` | 为本次会话禁用所有技能和斜杠命令 | `claude --disable-slash-commands` |

### 代理与子代理

| 标志 | 说明 | 示例 |
|------|------|------|
| `--agent` | 为本次会话指定代理 | `claude --agent my-custom-agent` |
| `--agents` | 通过 JSON 动态定义自定义子代理 | `claude --agents '{"reviewer":{"description":"...","prompt":"..."}}' ` |
| `--teammate-mode` | 设置队友显示方式：`auto`（默认）/`in-process`/`tmux` | `claude --teammate-mode in-process` |

### 工作目录与工作区

| 标志 | 说明 | 示例 |
|------|------|------|
| `--add-dir` | 添加额外工作目录供 Claude 访问 | `claude --add-dir ../apps ../lib` |
| `--worktree`, `-w` | 在隔离的 git worktree 中启动 Claude | `claude -w feature-auth` |
| `--ide` | 若恰好有一个有效 IDE 可用，启动时自动连接 | `claude --ide` |
| `--chrome` | 启用 Chrome 浏览器集成 | `claude --chrome` |
| `--no-chrome` | 为本次会话禁用 Chrome 浏览器集成 | `claude --no-chrome` |

### MCP 配置

| 标志 | 说明 | 示例 |
|------|------|------|
| `--mcp-config` | 从 JSON 文件或字符串加载 MCP 服务器 | `claude --mcp-config ./mcp.json` |
| `--strict-mcp-config` | 仅使用 `--mcp-config` 指定的 MCP 服务器，忽略其他配置 | `claude --strict-mcp-config --mcp-config ./mcp.json` |

### 设置加载

| 标志 | 说明 | 示例 |
|------|------|------|
| `--settings` | 加载额外设置文件（JSON 文件路径或 JSON 字符串） | `claude --settings ./settings.json` |
| `--setting-sources` | 指定加载的设置来源（`user`/`project`/`local`，逗号分隔） | `claude --setting-sources user,project` |
| `--plugin-dir` | 仅为本次会话从目录加载插件 | `claude --plugin-dir ./my-plugins` |

### 打印（非交互）模式

以下标志需配合 `-p` / `--print` 使用：

| 标志 | 说明 | 示例 |
|------|------|------|
| `--print`, `-p` | 打印响应后退出，不进入交互模式 | `claude -p "query"` |
| `--output-format` | 输出格式：`text`/`json`/`stream-json` | `claude -p "query" --output-format json` |
| `--input-format` | 输入格式：`text`/`stream-json` | `claude -p --input-format stream-json` |
| `--include-partial-messages` | 在输出中包含部分流事件（需 `stream-json` 格式） | `claude -p --output-format stream-json --include-partial-messages "query"` |
| `--json-schema` | 要求输出与指定 JSON Schema 匹配的验证 JSON | `claude -p --json-schema '{"type":"object",...}' "query"` |
| `--max-turns` | 限制代理最大轮次，达到上限时以错误退出 | `claude -p --max-turns 3 "query"` |
| `--max-budget-usd` | 设置 API 调用的最大费用上限（美元） | `claude -p --max-budget-usd 5.00 "query"` |
| `--no-session-persistence` | 禁用会话持久化，会话不保存到磁盘 | `claude -p --no-session-persistence "query"` |

### 远程与网络

| 标志 | 说明 | 示例 |
|------|------|------|
| `--remote` | 在 claude.ai 上创建新的网络会话 | `claude --remote "修复登录 Bug"` |
| `--remote-control`, `--rc` | 启动启用了 Remote Control 的交互式会话 | `claude --remote-control "My Project"` |
| `--teleport` | 在本地终端中恢复网络会话 | `claude --teleport` |

### 调试与日志

| 标志 | 说明 | 示例 |
|------|------|------|
| `--debug` | 启用调试模式，可选类别过滤 | `claude --debug "api,mcp"` |
| `--verbose` | 启用详细日志，显示完整逐轮输出 | `claude --verbose` |
| `--version`, `-v` | 输出版本号 | `claude -v` |
