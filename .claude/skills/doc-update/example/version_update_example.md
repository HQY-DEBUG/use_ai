# doc-update 使用示例

## 场景：对已有文档执行 `/doc-update` 更新

---

### 更新前的文档

```markdown
# DMA 驱动说明

> 文件：`dma_driver.md`  版本：v1.1  日期：2026-01-15

# 修改记录

| 版本   | 日期         | 修改内容             |
|------|------------|------------------|
| v1.1 | 2026-01-15 | 补充错误码说明        |
| v1.0 | 2026-01-01 | 创建文档             |

---

（正文略）
```

---

### 执行命令

```
/doc-update doc/dma_driver.md v1.2 新增多通道并发传输说明
```

---

### 更新后的文档

```markdown
# DMA 驱动说明

> 文件：`dma_driver.md`  版本：v1.2  日期：2026-03-23

# 修改记录

| 版本   | 日期         | 修改内容             |
|------|------------|------------------|
| v1.2 | 2026-03-23 | 新增多通道并发传输说明  |
| v1.1 | 2026-01-15 | 补充错误码说明        |
| v1.0 | 2026-01-01 | 创建文档             |

---

（正文略）
```

> 说明：版本记录仍为 3 条，未超出上限，无需删除旧记录。

---

### 场景二：超出 3 版本上限时

若已有 3 条记录，再执行 `/doc-update` 新增 v1.3 后：

```markdown
| 版本   | 日期         | 修改内容             |
|------|------------|------------------|
| v1.3 | 2026-03-23 | 调整章节结构          |
| v1.2 | 2026-03-23 | 新增多通道并发传输说明  |
| v1.1 | 2026-01-15 | 补充错误码说明        |
```

> 说明：v1.0（最旧）被自动移除，保持最多 3 条。

---

### 格式问题示例

若文档缺少标准文件头，`/doc-update` 将报告：

```
== 文档格式检查报告：dma_driver.md ==

❌ 问题项：2 项

  [行 1] 文件头：缺少版本信息行（> 文件：... 版本：... 日期：...）
  [行 5] 修改记录：表格缺少 "修改内容" 列

建议：按标准模板补充文件头后重新执行 /doc-update
```
