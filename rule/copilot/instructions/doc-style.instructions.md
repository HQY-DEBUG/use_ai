# 文档版本控制规范

## 文档头模板

每个 `.md` 文件**正文第一行**为一级标题，紧跟版本信息块（`>` 引用格式），再接修改记录表，用 `---` 与正文分隔：

```markdown
# 文档标题

> 版本：v1.0　日期：YYYY/MM/DD

# 修改记录

| 版本   | 日期         | 修改内容           |
|------|------------|------------------|
| v1.1 | YYYY/MM/DD | 补充…             |
| v1.0 | YYYY/MM/DD | 创建文档            |

---

（正文内容）
```

## 修改记录规则

- 最新版本排在**最前面**（倒序），仅保留最近 **3 个版本**（超出则删除最旧行）
- 版本号递增：`v1.0 → v1.1 → v1.2`；重大重构升主版本 `v1.x → v2.0`
- 描述使用**动词开头**的中文祈使句：✅ "新增…" / ❌ "新增了…"
- 文档头 `版本` 字段与修改记录表最新版本号**必须一致**
- 日期格式统一为 `YYYY/MM/DD`，禁止使用 `YYYY-MM-DD`

## 完整示例

```markdown
# SPI 主机控制器设计说明

> 版本：v1.2　日期：2026/03/27

# 修改记录

| 版本   | 日期         | 修改内容                  |
|------|------------|-------------------------|
| v1.2 | 2026/03/27 | 新增四线模式时序说明            |
| v1.1 | 2026/03/20 | 补充寄存器映射表              |
| v1.0 | 2026/03/15 | 创建文档                   |

---

（正文内容）
```

## 常见错误

```markdown
// ❌ 错误：日期格式错误
> 版本：v1.2　日期：2026-03-27

// ❌ 错误：版本头与记录表不一致
> 版本：v1.1
| v1.2 | 2026/03/27 | 新增… |   ← 头部还是 v1.1

// ❌ 错误：修改记录超过3行未清理
| v1.4 | 2026/03/27 | … |
| v1.3 | 2026/03/20 | … |
| v1.2 | 2026/03/15 | … |
| v1.1 | 2026/03/10 | … |   ← 应删除此行
```
