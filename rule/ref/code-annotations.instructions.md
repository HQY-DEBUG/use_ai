# 代码通用标注规范

## 新增代码标注

新增代码须在行尾或上方注释标注**版本与日期**：

```c
// v1.2 2026/03/17 新增：增加超时重传逻辑
if (timeout_cnt > MAX_RETRY) {
    retry();
}
```

Python / MATLAB：
```python
# v1.2 2026/03/17 新增：增加超时重传逻辑
```

## 修改代码标注

修改已有代码须在修改处标注版本与日期：

```c
// v1.2 2026/03/17 修改：超时阈值从固定值改为可配置参数
if (timeout_cnt > g_max_retry) {
    retry();
}
```

## 废弃代码标注

废弃代码须注明废弃版本、原因、计划清理版本，保留 **3 个版本**后清理：

```c
// [废弃] v1.2 2026/03/17 旧版超时逻辑，已由 retry() 替代，v1.5 后清理
// void timeout_old() { ... }
```

## 影响范围说明

修改已有函数或接口时，须在提交信息或注释中说明对其他模块的**影响范围**。
