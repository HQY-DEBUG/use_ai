# C/C++ 代码风格规范

## 缩进

- 使用 **4个空格** 缩进，禁止使用 Tab
- 大括号风格采用 K&R 风格（左括号不换行）

```c_cpp
// ✅ 正确
void foo() {
    if (x) {
        do_something();
    }
}
```

## 注释规范

- 函数头使用 Doxygen 风格（中文）：
  ```c_cpp
  /**
   * @brief  函数简要说明
   * @param  param1  参数1说明
   * @param  param2  参数2说明
   * @return 返回值说明
   * @note   注意事项（可选）
   */
  ```
- 行内注释使用 `//`，块注释使用 `/* */`
- 注释内容使用**中文**
- 新增代码标注版本与日期：`// vX.Y YYYY/MM/DD 新增：说明`
- 分隔式注释（区块标题）统一使用 `// ---- 内容 ----//`，不使用 `/* --- 内容 --- */`：
  ```c_cpp
  // ❌ 避免
  /* --- 校验 FrameTail --- */

  // ✅ 正确
  // ---- 校验 FrameTail ----//
  ```

## 文件头模板

每个 `.c` / `.h` 文件顶部必须包含：

```c_cpp
/*
 * 文件 : xxx.c
 * 描述 : 模块简要说明
 * 版本 : v1.0
 * 日期 : YYYY/MM/DD
 *
 * 修改记录（最新版本在最前）:
 *  ver  who      date       modification
 * ----- -------  ---------- ---------------------------------
 * 1.0   xxx      YY/MM/DD   创建文件
 */
```

## 命名规范

- 变量/函数：小写下划线风格，如 `recv_buf`、`init_dma()`
- 宏定义：全大写下划线，如 `MAX_BUF_SIZE`
- 全局变量加 `g_` 前缀，如 `g_tx_count`
