# doc/ 文档版本控制规范

## 文档头模板

每个 `doc/` 下的 `.md` 文件，**正文第一行**为一级标题，其后紧跟版本信息块（`>` 引用格式）：

```markdown
# 文档标题

> 文件：`文件名.md`
>
> 版本：v1.0
>
> 日期：YYYY/MM/DD
>
> 面向当前工程：（可选，描述文档适用范围）

# 修改记录

| 版本 | 日期 | 修改内容 |
|---|---|---|
| v1.0 | YYYY/MM/DD | 创建文档：简要说明初始内容 |
```

> **最新版本排在表格最前面**（倒序），方便直接看到最新改动。

## 修改记录规则

- 每次实质性修改（新增章节、结构调整、内容补充）必须**在表格最前面插入新行**（倒序，最新在上）
- 版本号递增：`v1.0 → v1.1 → v1.2 ...`，重大重构可升主版本 `v1.x → v2.0`
- 修改内容描述使用**动词开头**的中文祈使句，如"新增…"、"补充…"、"修正…"、"调整…"、"删除…"
- 修改记录表放在文档头版本块之后，正文内容之前，用 `---` 分隔线与正文分开

## 完整示例

```markdown
# FreeRTOS 编码特性与入门

> 文件：`FreeRTOS编码特性与入门.md`
>
> 版本：v1.2
>
> 日期：2026-03-12
>
> 面向当前工程：Zynq PS + lwIP + UDP 协议 + DMA 中断 + `list_mem` 列表执行

# 修改记录

| 版本 | 日期 | 修改内容 |
|---|---|---|
| v1.2 | 2026-03-12 | 调整结构：将工程相关内容收拢至文末独立章节 |
| v1.1 | 2026-03-12 | 补充事件组、软件定时器、`FromISR` API 说明 |
| v1.0 | 2026-03-12 | 创建文档：介绍 FreeRTOS 基本概念、任务、调度 |

---

（正文内容）
```

## 注意事项

- 文档头的`版本`字段与修改记录表的最新版本号**必须保持一致**
- `日期`字段格式统一为 `YYYY/MM/DD`
- 文件名本身不含版本号，版本信息完全在文档内维护
