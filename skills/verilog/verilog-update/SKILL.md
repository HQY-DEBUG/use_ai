# 更新 Verilog 模块文件头

参数：$ARGUMENTS

## 操作步骤

1. 从参数解析：文件路径、新版本号（如 `v1.1`）、修改说明
2. 读取目标文件，定位文件头区域（`/***...`开头到第一个 `endmodule` 之前）
3. 获取当前日期（PowerShell）：`Get-Date -Format "yyyy/MM/dd"`
4. 更新以下字段：
   - `// Version    :` → 新版本号
   - `// Date       :` → 今天日期
   - 在 Modify 表格**最前面**插入新修改记录行
   - 若已有 3 条修改记录，删除最旧一条，保留最近 3 条
5. 使用 Edit 工具精确替换文件头，**不修改模块正文**
6. 输出更新前后的文件头对比
7. 提醒用户执行 git commit

## 文件头修改示例

修改前：
```
// Version    : v1.0
// Date       : 2026/01/01
// ...
// Modify:
// version       date       modify
// --------    -----------  ------------------------------------------------
//  v1.0        2026/01/01  创建文件
```

修改后（新增 v1.1）：
```
// Version    : v1.1
// Date       : 2026/03/26
// ...
// Modify:
// version       date       modify
// --------    -----------  ------------------------------------------------
//  v1.1        2026/03/26  <修改说明>
//  v1.0        2026/01/01  创建文件
```

## 注意事项

- 修改记录最多保留 **3 条**，超出时删除最旧一条
- 仅修改文件头，不改动模块端口、信号声明、逻辑代码
- 修改说明使用动词开头的中文祈使句（"新增…"、"修正…"、"优化…"）
- 操作前先显示当前文件头，确认后再执行修改
