# C/C++ 代码审查

审查目标：$ARGUMENTS

## 操作步骤

1. 读取指定文件（若未指定，读取当前目录下所有 `*.c` / `*.h` / `*.cpp` / `*.hpp`）
2. 逐项检查以下规范，记录每处违规的**行号**和**问题描述**
3. 按检查项汇总输出审查报告
4. **仅输出报告，不修改任何文件，不执行 git 提交**

## 检查项清单

### 文件头
- [ ] 顶部包含标准文件头（文件、描述、版本、日期、修改记录）
- [ ] 修改记录最多保留 3 个版本，最新在最前

### 缩进与括号
- [ ] 使用 4个空格 缩进，无 Tab 字符（`\t`）
- [ ] 大括号采用 K&R 风格（左括号不换行）

### 函数调用
- [ ] 函数调用写在同一行，无跨行参数拆分

### 注释规范
- [ ] 行内注释使用 `//`，无行内 `/* */`
- [ ] 区块标题使用 `// ---- 内容 ----//`，无 `/* ---- 内容 ---- */`
- [ ] 注释内容使用中文
- [ ] 函数头使用 Doxygen 风格（`@brief` / `@param` / `@return`）

### 命名规范
- [ ] 变量、函数使用小写下划线
- [ ] 宏定义使用全大写下划线
- [ ] 全局变量使用 `g_` 前缀
- [ ] 布尔变量使用 `is_`、`has_`、`can_` 前缀

### 废弃代码
- [ ] 废弃代码有注释说明原因，含废弃日期和替代方案

### Qt 专项（适用 .cpp / .h）
- [ ] 继承 `QObject` 的类第一行包含 `Q_OBJECT`
- [ ] 构造函数接受 `QObject *parent = nullptr` 并传给基类
- [ ] 跨线程操作使用信号槽，无从子线程直接操作控件
- [ ] 信号名使用动词过去式，槽名使用 `on` + 信号名

## 输出格式

```
== C/C++ 规范审查报告：<文件名> ==

✅ 通过项：X 项
❌ 违规项：Y 项

违规详情：
  [行 NN] <检查项>：<问题描述>
  [行 NN] <检查项>：<问题描述>
  ...

建议：<总体改进建议>
```
