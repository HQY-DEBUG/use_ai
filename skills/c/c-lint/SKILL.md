# C/C++ 静态质量审查

参数：$ARGUMENTS

## 与 /c-review 的区别

| | `/c-review` | `/c-lint` |
|--|-------------|-----------|
| 关注点 | 编码规范（格式/命名/注释） | 逻辑质量（潜在 bug/资源管理） |
| 输出 | 规范违反列表 | 问题分级报告 |

## 操作步骤

1. 读取目标文件
2. 按以下分类逐项检查，记录问题
3. 输出分级报告

## 检查项

### 🔴 高风险
- 指针使用前未检查 NULL（`malloc`/`fopen` 返回值未校验）
- 数组越界风险（固定长度缓冲区 + 外部输入长度）
- `sprintf` / `strcpy` 使用（应改用 `snprintf` / `strncpy`）
- 整数溢出（有符号与无符号混用、位移超出类型宽度）
- `free` 后未置 NULL（悬空指针）

### 🟡 中风险
- 文件/socket/内存分配后存在未关闭/释放的路径
- 函数返回值被忽略（尤其 `write`/`read`/`send`/`recv`）
- 局部变量未初始化即使用
- `switch` 缺少 `default` 分支
- 全局变量在多处写入（线程安全风险）

### 🟢 低风险 / 建议
- 魔法数字未定义为宏
- 函数超过 80 行（建议拆分）
- 嵌套深度超过 4 层

## 输出格式

```
文件：xxx.c
────────────────────────────────
🔴 第 42 行：malloc 返回值未检查 NULL，若内存不足将导致空指针解引用
   `buf = malloc(size);`
   建议：`if (buf == NULL) { log_error(...); return -1; }`

🟡 第 78 行：fopen 打开的文件句柄在错误路径未关闭
   ...

统计：🔴 高风险 N 处　🟡 中风险 M 处　🟢 建议 L 处
```

## 注意事项

- 不修改任何文件
- 仅做静态分析，不运行代码
- 对 Vitis/嵌入式场景：`malloc` 使用本身需谨慎，发现时额外提示
