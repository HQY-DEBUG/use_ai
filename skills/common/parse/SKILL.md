# 需求解析与执行

用户输入：$ARGUMENTS

## 执行步骤

### 第一步：解析需求

仔细阅读用户输入，结合当前项目上下文，将其拆解为具体可执行的子任务。

输出格式如下（严格按此格式，不要省略）：

---
**需求解析结果**

原始输入：`<用户输入原文>`

拆解为以下任务：
1. **[文件/模块]** 具体操作描述
2. **[文件/模块]** 具体操作描述
...

执行顺序：按上述编号顺序执行
---

### 第二步：逐步执行

按解析出的任务列表，逐条执行：
- 每完成一个任务，标注 `✅ 任务N 完成`
- 涉及文件修改的，先读取再修改
- 所有实质性修改完成后执行 git commit

### 注意事项

- 解析时结合已打开的文件和项目结构理解上下文
- 如果输入有歧义，在解析结果中列出假设条件
- 不要添加用户未要求的额外功能
- 保持最小修改原则
